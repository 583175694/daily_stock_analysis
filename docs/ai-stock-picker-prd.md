# AI 选股 PRD

## 1. 文档定位

本文档只回答产品问题：

- 为什么做 AI 选股
- 给谁用
- 要做什么
- 各版本边界是什么
- 如何判断版本验收通过

实现分层、目录结构、API、数据表、开发顺序、技术 TODO 统一放在：

- `docs/ai-stock-picker-design.md`

## 2. 产品背景

当前项目已经具备较完整的股票分析底座：

- 对指定股票做单股分析
- 做大盘复盘
- 提供 Agent 问股与深度研究
- 支持历史回测
- 管理持仓与组合风险

当前缺的不是“分析能力”，而是“机会发现入口”。

用户仍需要先自己决定分析哪些股票，系统更像“分析器”，还不是“发现器”。AI 选股的目标，是让系统先从股票池中筛出候选，再给出解释、后续动作建议和结果跟踪能力。

## 3. 问题定义

当前核心问题：

- 用户每天仍要手工先挑股票
- 自选池一多，很难快速定位值得优先看的少数标的
- 现有系统强在单股分析，弱在上游批量筛选与排序
- 回测能验证分析结果，但还不能系统验证“候选股是否有效”
- 缺少“市场环境 -> 候选股 -> 深度分析 -> 后验跟踪”的闭环

## 4. 产品目标

### 4.1 核心目标

- 让系统具备从股票池中自动筛选候选股的能力
- 让候选结果可解释，而不是黑盒推荐
- 让候选结果可回看、可跟踪、可验证
- 让选股能力能与单股分析、Agent、通知、回测、组合模块形成闭环

### 4.2 业务目标

- 提升用户每日使用频率
- 提升从“跑分析”到“采取行动”的转化效率
- 提升批量场景下的决策效率
- 为后续模板化策略、参数化权重和组合联动打基础

### 4.3 非目标

- 第一阶段不追求全自动实盘交易
- 第一阶段不做复杂量化回测框架重构
- 第一阶段不做因子研究平台
- 第一阶段不做社交型选股社区

## 5. 目标用户

### 5.1 核心用户

- 已在使用本项目做日常股票分析的活跃用户
- 拥有稳定自选股池，需要每日筛重点关注标的的用户
- 关注 A 股、港股、美股至少一个市场的用户

### 5.2 典型用户类型

#### 用户类型 A：短线交易者

- 更关注趋势、量能、情绪、板块轮动
- 更需要快速筛出强势候选
- 对“为什么入选”和“什么时候观察/止损”敏感

#### 用户类型 B：波段交易者

- 更关注趋势启动、回调承接、景气变化
- 更需要候选排序和风险解释
- 希望结合市场环境判断是否值得跟踪

#### 用户类型 C：组合管理者

- 已有真实持仓
- 更关注新候选和现有仓位是否存在重复暴露
- 希望候选结果能联动组合视角

## 6. 核心价值主张

AI 选股不是替代单股分析，而是在单股分析之前新增“机会发现层”。

用户最终获得的价值：

- 系统先帮我找机会
- 系统告诉我为什么这些票值得看
- 系统告诉我主要风险是什么
- 系统支持从候选直接进入后续动作
- 系统能持续验证自己的选股效果

## 7. 核心使用场景

### 场景 1：每日盘后选股

用户运行选股任务，系统从指定股票池中筛出 Top N 候选，并给出入选理由、风险点和下一步建议。

### 场景 1A：先选板块，再选个股

用户先选择一个或多个关注板块，再在对应板块股票范围内运行 AI 选股，系统输出该板块下更值得关注的候选股。

### 场景 2：从候选到单股深挖

用户对候选中的 1 到 3 只股票发起单股详细分析或 Agent 追问。

### 场景 3：从候选到观察池

用户把暂时不买、但值得继续跟踪的股票加入观察池或自选列表。

### 场景 4：验证最近一段时间哪种模板更有效

用户查看不同选股模板的命中率、收益率和回撤表现，决定是否继续使用某个模板。

### 场景 5：结合持仓做增量决策

用户已有持仓，系统输出候选时同时提示行业暴露、重仓重合或风险重复问题。

## 8. 产品原则

- 先筛选，再解释，不做纯黑盒推荐
- 先做可回看、可追踪、可验证，再做更复杂的 AI 增强
- 先内置模板，后开放参数化和自然语言输入
- 先服务已有用户的日常流程，再考虑更广泛的外部用户
- AI 负责解释和复核，不完全替代结构化规则和结构化评分

## 9. 功能范围总览

AI 选股能力分三层：

- 候选生成
- 候选解释
- 候选验证

对应的产品能力：

- 股票池选择
- 板块选择
- 选股模板
- 手动和定时运行
- 结果列表与详情
- 候选解释
- 一键后续动作
- 历史记录与跟踪
- 效果统计
- 组合联动

## 10. 版本规划

开发顺序统一按 `V1 -> V2 -> V3 -> V4 -> V5` 推进，不再使用 `M0/M1/M2` 这类独立里程碑命名。

### 10.1 V1：MVP

目标：

- 做出第一版可用、可解释、可保存结果的 AI 选股能力

范围：

- 股票池选择
- 选股模板选择
- 手动运行一次选股
- 候选列表展示
- 候选详情展示
- 一键发起单股分析
- 一键跳转 Agent 问股
- 一键加入自选股
- 历史任务与结果回看

不包含：

- 定时自动执行
- 结果后验跟踪
- 模板效果统计
- 自动通知
- 组合风险联动
- 用户自定义复杂股票池编辑器
- 全市场全量扫描

当前状态：

- 已完成
- 已验收通过

### 10.2 V2：验证增强版

目标：

- 让选股结果具备后验验证和复盘能力

范围：

- 板块优先选股
- 参数化模板
- 候选股后续表现跟踪
- 模板效果统计
- 胜率、收益率、回撤等指标展示
- 选股结果摘要通知

当前状态：

- 已实现
- 待于 2026-04-15 验证
- 验证通过后再提交代码

### 10.3 V3：核心完善版

目标：

- 在扩展入口前，先把选股核心做得更稳定、更可解释、更可验证

范围：

- 数据与口径稳定化
- 模板筛选质量收口
- 评分解释一致性增强
- 后验评估与模板统计补验
- 板块层增强，为后续 `AI 选板块` 做准备

当前状态：

- 第二阶段代码已完成
- 已完成历史回放 / 模拟验证，用于提前验证后验计算与 `000300` 基准补数链路
- 尚未完成真实自然样本下的端到端后验验证与 `000300` 基准验证
- 仍需在 `5/10/20` 日窗口自然成熟后，按真实任务链路补做候选详情、模板统计、数据库落库与基准稳定性的 E2E 验证

### 10.4 V4：精度增强版

目标：

- 在核心可信度收口后，继续提升 AI 选股的实战精度、可执行性与环境适配能力

范围：

- 明确买入、卖出、持有、止盈止损规则
- 增加按市场环境、模板、排名层、信号强弱等维度的分层统计
- 增加市场环境过滤与模板适配
- 增强相对强弱、板块地位、流动性、事件强度等有效因子
- 增加真实交易约束与成本建模
- 增加 AI 二次复核、风险否决与信号置信度输出

当前状态：

- 待开发

实施建议：

- V4 应继续作为单一版本名义推进，但实施上拆分为 `V4.1` 与 `V4.2` 两个阶段，避免把“可信度收口”和“专业化扩张”混在一次交付中

#### V4.1：可信执行版

目标：

- 先让当前选股能力从“值得看”升级为“规则更完整、评估更可信、结果更接近可执行”

范围：

- 补完 `5/10/20` 日窗口后验计算、模板统计与 `000300` 基准稳定性验证
- 为模板补齐 alpha 假设、适用市场环境、失效环境与排除条件
- 增加最小可用的市场环境标签与模板过滤 / 降权
- 落地统一的买入、卖出、持有、止损、止盈与超时退出规则
- 引入最小版真实交易约束：滑点、跳空、低流动性与不可成交情形
- 增加按市场环境、模板、排名层、信号强弱的基础分层统计

说明：

- `V4.1` 的核心不是扩更多入口，而是先把当前 A 股选股链路的可信度、可执行性与解释边界收口
- 若 `V3` 的后验补验与基准稳定性未完成，不应把 `V4.1` 视为已具备验收前提

#### V4.2：专业增强版

目标：

- 在 `V4.1` 可被稳定验证后，在**不新增重型研究底座**的前提下，继续提升当前选股链路的单票区分度、信号质量与置信度表达

范围：

- 增强相对强弱、板块地位、流动性质量、事件强度等有效因子
- 增加 AI 二次复核、反例挑战、风险否决与置信度输出
- 在现有数据底座可支持的范围内，补充更细的样本外、滚动与风险调整评估
- 继续提高单票与模板级别的解释一致性、失效识别与排序稳定性

说明：

- `V4.2` 不再承接板块研究体系、行业 / 事件标签体系、多基准研究框架、组合研究框架与研究副驾主线，这些重叠部分统一收拢到 `V5`
- `V4.2` 的职责边界是：在当前底座上把“单票选得更准、解释更稳、置信度更可信”继续做深，而不是升级为完整研究平台
- `V4.2` 默认优先做 A 股；港股、美股要进入同等专业深度前，仍需先解决数据口径与外部信息源不足问题

### 10.5 V5：A 股专业深化版

目标：

- 在 `V4` 完成可信度与可执行性收口后，把 A 股 AI 选股继续推进到更接近“最终专业形态”的研究与策略框架

范围：

- AI 选板块
- 标准化行业 / 概念 / 事件 / 宏观环境数据体系
- 板块研究与轮动判断体系
- 模板版本化、策略家族化与因子贡献归因
- 多基准、多环境、风险调整后的评估体系
- AI 研究副驾：公告 / 新闻 / 研报摘要、多空论点、反例挑战、模板失效诊断
- 优先基于公开免费源与免费但有限额 / 有限权源完成 A 股专业深化，不默认依赖高价销售报价型数据源

说明：

- `V5` 仍然保留 `AI 选板块`，因为它依赖板块研究、行业分类、市场环境与候选板块收敛能力，天然更适合放在专业深化阶段统一推进
- `V5` 默认优先使用 `巨潮资讯 / 中证指数 / 国证指数 / 国家统计局 / 人民银行 / Shibor / 中国货币网` 等公开免费源，以及 `Tushare Pro / AkShare / efinance / pytdx / Baostock / 搜索服务` 等免费但有限额 / 有限权源
- 原先与 `V4.2` 存在重叠的专业深化内容，如板块研究体系、模板版本化、策略家族化、因子贡献归因、多基准 / 多环境评估与 AI 研究副驾，统一以 `V5` 为主版本承接
- 原先偏工作流放大器的能力，如 `自然语言策略输入`、`定时自动选股`、`盘前/盘后任务计划`、`组合暴露联动`、`候选与持仓联动建议` 等，顺延到 `V5` 之后的后续版本，避免在专业深化阶段再次把范围拉散

当前状态：

- 待开发

## 11. 策略输入模式演进

### V1：内置模板

- 系统预置模板
- 用户只选择，不自定义模板结构
- 保证稳定、可解释、可回看

### V2：参数化模板

- 用户基于内置模板做受控调参
- 调整的是参数，不是模板结构本身

### V2：板块优先选股

- 用户可先选择一个或多个板块，再在板块内选股
- 板块层用于扩大覆盖范围，但不直接等同于全量市场扫描

### V3：核心完善优先级

- 先稳定数据口径、模板质量、解释一致性与后验评估
- 不在该阶段引入会显著放大不稳定性的自动化入口或自由输入入口

### V4：精度增强优先级

- 先补完整交易规则、分层统计、市场环境过滤、有效因子与真实约束
- AI 在该阶段优先做复核、否决、风险提示和置信度表达，不直接替代结构化排序

### V5：自然语言策略输入

- 用户通过自然语言描述选股诉求
- 系统先归一化为结构化模板，再进入执行层

### V5：AI 选板块

- 系统先基于结构化规则收敛候选板块
- 再由 AI 对少量候选板块做解释、排序或推荐
- 最终进入板块内 AI 选股，而不是直接对全市场做黑盒决策

当前产品结论：

- V1 只做内置模板
- 不允许 LLM 直接决定排名
- 不允许 LLM 在 V1 自由定义策略结构

## 12. V1 功能需求

### 12.1 股票池选择

要求：

- 用户可选择一个可执行的股票池作为本次选股输入
- V1 至少支持当前 `STOCK_LIST`
- 若有预设池，需明确池名称与适用市场

### 12.2 选股模板选择

要求：

- 用户可在内置模板中选择一个模板执行
- 模板展示应包含名称、说明、适用市场、风险提示

### 12.3 手动运行选股

要求：

- 用户可从页面手动发起一次选股任务
- 任务具备明确的运行状态和结果返回

### 12.4 候选结果列表

要求：

- 展示候选股票列表
- 展示排序、总分、关键解释、主要风险
- 支持查看结果对应任务和模板信息

### 12.5 候选详情

要求：

- 展示单只候选的结构化评分
- 展示 AI 解释摘要
- 展示模板上下文与风险点

### 12.6 一键后续动作

要求：

- 支持从候选结果一键发起单股分析
- 支持一键跳转 Agent 问股
- 支持一键加入自选股或观察池

### 12.7 历史记录

要求：

- 可回看历史选股任务
- 可查看历史任务下的候选结果
- 可识别任务所使用的股票池、模板和时间

## 13. V2 功能需求

### 13.1 板块优先选股

- 支持用户先选择一个或多个板块，再在板块内执行 AI 选股
- 首版只支持用户手动选择 A 股行业板块
- 支持展示板块名称、板块说明、适用市场和覆盖股票数量
- 板块内仍然先走结构化过滤和结构化评分，再对 Top 候选做 AI 解释
- 不要求 V2 直接支持全量 A 股全市场扫描

### 13.2 参数化模板

- 支持在内置模板上做受控参数覆盖
- V2 首版只开放数量参数：`limit`、`ai_top_k`
- 参数范围需可校验、可展示、可回放

### 13.3 结果后验跟踪

- 跟踪候选股后续表现
- 首版固定使用 `5/10/20` 日窗口
- 支持按时间窗口回看命中率、收益率、超额收益、回撤

### 13.4 模板效果统计

- 支持不同模板的横向对比
- 首版重点覆盖 A 股候选
- V2 首版聚焦 A 股与固定 `5/10/20` 日窗口
- 默认横向比较全部模板，暂不额外开放市场/时间范围筛选

### 13.5 通知摘要

- 支持将结果摘要发送到已有通知渠道
- 首版提供可选通知开关，默认关闭

## 14. V3 功能需求

### 14.1 数据与口径稳定

- 固定交易日口径、板块口径、基准口径与异常数据处理口径
- 减少不同运行时间、不同数据源下的结果歧义

### 14.2 模板筛选质量收口

- 收紧严格命中与补位候选边界
- 复核过滤条件、关键分数权重与排序稳定性
- 优先提高候选质量，而不是先扩大入口

### 14.3 解释与评分一致性

- 候选解释需与评分拆解、技术快照、板块信息保持一致
- 风险点与观察点需支持后续调优、归因与复盘

### 14.4 后验评估补验

- 在 `5/10/20` 日窗口成熟后补验后验计算正确性
- 校验模板统计与候选明细一致性
- 单独验证 `000300` 基准稳定性

### 14.5 板块层增强

- 在稳定板块口径基础上增强板块强度、轮动与候选板块收敛能力
- 该层增强服务于后续 `AI 选板块`，不直接替代个股评分层

## 15. V4 功能需求

### 15.1 交易规则完整化

- 补齐买入、卖出、持有周期、止盈止损等规则
- 让候选结果从“值得看”升级为“可执行、可验证”

### 15.2 分层统计与分桶评估

- 支持按市场环境、模板、信号强弱、排名层、板块属性等维度拆分统计
- 避免只看整体均值而掩盖策略真实边界

### 15.3 市场环境过滤

- 支持根据指数趋势、板块扩散、成交环境与情绪状态做过滤或降权
- 避免模板在不适用的市场状态下被错误放大使用

### 15.4 有效因子增强

- 增加相对强弱、板块地位、流动性质量、事件强度等因子
- 优先补“能提高区分度”的因子，而不是堆叠更多弱相关特征

### 15.5 真实约束与成本建模

- 在评估与执行层引入滑点、跳空、流动性、不可成交情形等真实约束
- 降低“纸面表现很好、实盘执行一般”的偏差

### 15.6 AI 二次复核与置信度输出

- AI 优先对高分候选做复核、风险提示与一致性检查
- 输出信号置信度、超额收益概率或风险概率等概率化表达
- 置信度默认拆分为 `研究置信度` 与 `执行置信度`，避免把“逻辑成立”与“交易可执行”混为单一黑盒分数
- `研究置信度` 用于表达候选在给定模板、市场环境、评估窗口与基准下取得正收益或超额收益的结构化胜算
- `执行置信度` 用于表达候选在流动性、波动、跳空、涨跌停与成交约束下按预设规则完成交易执行的把握程度
- 首版允许输出综合置信度，但必须能回拆到 `研究置信度` 与 `执行置信度`
- 所有概率化表达必须绑定明确的评估窗口、比较基准、入场规则与退出规则，不允许脱离口径单独展示“高置信度”

### 15.7 置信度校准与样本外验证

- 所有概率化输出必须基于历史候选后验做分桶校准，不允许直接把 LLM 主观判断包装成概率
- 至少校验不同置信度分桶的真实命中率、平均超额收益与最大回撤是否随置信度提升而改善
- 至少完成一次样本外验证与滚动窗口验证，避免只在单一历史区间或单次回看结果上得出结论
- 首版默认围绕 A 股与固定 `5/10/20` 日窗口做校准，并绑定冻结后的规则版本、模板版本与市场环境标签
- 当可比样本不足、环境标签不稳定或校准结果失真时，只允许输出“样本不足 / 观察中”，不展示“高置信度”标签

### 15.8 模板假设与失效条件

- 每个模板必须补齐 alpha 假设、适用市场环境、失效环境、排除条件与主要风险来源
- 模板的买入、卖出、持有、止盈止损与超时退出规则必须与后验评估口径保持一致
- 候选详情需明确展示“为什么适用”“为什么可能失效”“什么情况下只观察不交易”
- 当候选触发模板失效条件、环境不匹配或执行约束明显恶化时，系统应支持否决、降权或明确标记“仅观察不交易”

### 15.9 V4 数据源边界

- `V4.1` 默认优先面向 A 股，尽量在现有行情、板块、资金流、财报摘要、分红、龙虎榜、筹码与新闻搜索能力上完成第一阶段收口
- `V4.2` 若要达到更专业的投研质量，需要补强正式行业分类、结构化公告 / 事件、研报 / 盈利预期修订、行业 / 风格指数与更完整的多基准序列
- 港股、美股当前可以作为行情与解释补充市场，但不应默认承诺与 A 股同等深度的因子、评估与组合联动能力

## 16. V5 功能需求

### 16.1 AI 选板块

- 支持系统先输出少量候选板块
- 支持对候选板块给出入选理由、风险提示和优先级
- 板块推荐应建立在结构化板块筛选之上，不直接依赖 LLM 黑盒扫描全市场

### 16.2 数据与因子体系深化

- 补齐行业、概念、事件、宏观环境等关键研究标签
- 优先使用公开免费源与免费但有限额 / 有限权源完成 A 股研究底座增强
- 支持把结构化数据沉淀为可回放、可比较、可归因的特征与标签

### 16.3 板块研究与策略对象深化

- 支持板块强度、轮动、持续性、龙头 / 中军 / 跟风识别
- 支持模板版本化、策略家族化与因子贡献归因
- 支持在不同市场环境下比较模板与板块研究效果

### 16.4 评估体系深化

- 支持多基准、多环境、风险调整后评估
- 支持 `MFE / MAE / 盈亏比 / 持有周期分布` 等更专业的分布型指标
- 支持对高置信度模板-环境组合做持续样本外与滚动验证

### 16.5 AI 研究副驾

- 支持公告 / 新闻 / 研报摘要压缩
- 支持候选股多空论点生成、反例挑战与模板失效诊断
- AI 仍然只做研究增强、解释与复核，不直接替代结构化主排序逻辑

### 16.6 V5 数据源策略

- 优先接入 `巨潮资讯 / 中证指数 / 国证指数 / 国家统计局 / 人民银行 / Shibor / 中国货币网`
- 优先复用已集成的 `Tushare Pro / AkShare / efinance / pytdx / Baostock / 搜索服务`
- 默认不要求 `Wind / iFinD / Choice / JQData / RQData` 作为 `V5` 的前置条件

### 16.7 顺延到 V5 之后的工作流能力

- `自然语言策略输入`
- `定时自动选股`
- `盘前 / 盘后任务计划`
- `组合暴露联动`
- `按市场环境动态调整模板`
- `候选与持仓的联动建议`

## 17. 页面需求

V1 页面至少覆盖：

- 运行入口
- 模板与股票池选择
- 任务列表
- 候选结果表
- 候选详情视图
- 一键后续动作

V2/V3/V4/V5 的页面可在此基础上继续扩展：

- 模板参数配置
- 效果统计面板
- 跟踪与复盘视图
- 计划任务管理
- 组合联动视图

## 18. 输出物定义

一次选股任务的核心输出：

- 任务元信息
- 候选列表
- 候选排序与分数
- 每只候选的关键解释
- 风险提示
- 可触发的后续动作

## 19. 核心用户流程

### 流程 A：手动选股

1. 用户进入 AI 选股页面
2. 选择股票池
3. 选择模板
4. 运行选股
5. 查看候选列表
6. 展开候选详情
7. 对候选执行后续动作

### 流程 B：结果回看

1. 用户进入历史任务列表
2. 选择某次历史任务
3. 查看当时的候选结果和解释

### 流程 C：加入观察池

1. 用户在候选详情中确认标的值得继续关注
2. 将该标的加入自选股或观察池

## 20. 核心指标

### 20.1 产品使用指标

- 选股任务运行次数
- 结果查看率
- 从候选发起后续动作的转化率

### 20.2 结果质量指标

- 候选后续命中率
- 平均收益率
- 最大回撤
- 模板间效果差异

### 20.3 系统效率指标

- 单次选股耗时
- 失败率
- 降级成功率

## 21. 非功能要求

- 排序逻辑应可解释、可回放
- 单一数据源失败不应直接拖垮整条链路
- 历史结果应可回看
- 输出结构应能支撑后续统计与验证

## 22. 验收标准

### 22.1 V1 验收标准

- 用户可完成一次手动选股
- 系统可返回候选列表和候选详情
- 用户可从结果进入后续分析动作
- 历史任务可回看
- 结果具备基本可解释性

当前状态：

- 已验收通过

### 22.2 V2 验收标准

- 能查看候选结果后验表现
- 能比较不同模板效果
- 能输出统计和通知摘要

### 22.3 V3 验收标准

- 模板筛选逻辑更稳定，严格命中与补位边界清晰
- 评分解释、技术快照、板块信息可对齐、可复盘
- `5/10/20` 后验结果、模板统计与 `000300` 基准稳定性完成补验

### 22.4 V4 验收标准

- 交易规则、持有规则与退出规则明确且可回放
- 每个模板的 alpha 假设、适用环境、失效环境与排除条件可查看、可回放、可用于复盘归因
- 能按市场环境、信号层次、模板与排名层查看分层统计
- 候选结果能体现更真实的执行约束、风险提示与置信度
- 置信度默认拆分为 `研究置信度` 与 `执行置信度`，并绑定明确的评估窗口、比较基准、入场规则、退出规则与规则版本
- 概率化输出已基于历史后验完成基础分桶校准；高置信度桶的真实命中率与预测概率偏差原则上不超过 `10` 个百分点，且表现不低于中低置信度桶
- 至少完成一次样本外验证与滚动窗口验证，验证结论可按市场环境查看，而不只展示整体均值
- `高置信度` 标签仅在对应模板-环境组合的可比样本不少于 `50`、主评估窗口平均超额收益为正且校准通过后才允许展示；不满足条件时只允许展示“中性 / 观察中 / 样本不足”

### 22.5 V5 验收标准

- `AI 选板块` 能建立在结构化候选板块收敛之上稳定运行
- 已接入一批可复用的 A 股行业 / 指数 / 公告 / 宏观数据源，且这些数据能进入统一标签或特征链路
- 板块研究、模板版本化、因子贡献归因与多基准 / 多环境评估至少完成最小可用闭环
- AI 研究副驾可输出公告 / 新闻 / 研报摘要、反例挑战与模板失效诊断，并能与结构化评分结果对齐
- `V5` 默认不依赖销售报价型数据源也能完成首版验收；若某些能力因免费数据源不足被降级，必须在验收说明中明确边界

## 23. 当前建议

- `PRD` 保持稳定，聚焦产品边界和版本目标
- 技术实现、开发进度、TODO、决策收口统一维护在 `docs/ai-stock-picker-design.md`

当前阶段结论：

- V3 前优先补强选股核心，而不是继续扩大入口
- V4 优先补“从能信到更准”的能力，包括交易规则、分层统计、市场环境过滤、有效因子与真实执行约束
- `V4.2` 聚焦当前底座上的单票精度与置信度增强；与研究平台升级相关的重叠部分统一收拢到 `V5`
- `AI 选板块` 保留在 `V5`，并和 A 股专业深化、板块研究、数据源升级一起推进
- `定时自动选股`、`自然语言策略输入`、`盘前 / 盘后任务计划`、`组合暴露联动` 等工作流放大器顺延至 `V5` 之后，避免在专业深化阶段打散主线

## 24. V5 之后的专业化扩展方向

说明：

- 本章节不做排期
- 本章节不表示全部都要做
- 本章节用于回答“在 `V5` 完成 A 股专业深化后，若继续往更专业、更完整的投研产品演进，应重点补哪些能力”

### 24.1 对当前方案的判断

判断：

- 当前方案是合理的
- 当前方案更接近“闭环验证版”而不是“最终专业版”

理由：

- 当前版本优先验证“选股 -> 解释 -> 后验 -> 统计 -> 通知”的闭环是否成立
- 当前版本优先控制 token 成本、交互复杂度与实现风险
- 当前版本已经采用“结构化评分为主、AI 解释为辅”的正确方向
- 但当前板块口径、后验口径、组合联动、研究深度和执行深度仍偏首版实现

结论：

- 当前路线是对的
- 但如果产品继续演进，下一阶段重点不应只是继续加页面功能，而应升级为更完整的投研能力体系

### 24.2 V5 之后的演进原则

- AI 不直接替代结构化排序逻辑
- 结构化规则、因子与风控仍应是选股主干
- AI 更适合做信息压缩、研究归纳、反例挑战和解释增强
- 新增能力优先提升“研究质量、评估质量、组合质量”，而不是优先增加更多入口
- 若某项能力无法沉淀为可回放、可比较、可归因的数据结构，则不应直接进入核心链路

### 24.3 建议补强的能力层

#### 1. 数据层

目标：

- 从“有数据可跑”升级为“有研究底座可复用”

建议方向：

- 标准化行业体系：申万、中信等正式行业分类
- 标准化概念体系：题材概念、产业链、风格标签
- 财务因子：营收、利润、ROE、现金流、估值分位
- 资金因子：主力资金、北向资金、融资融券、成交额分位
- 事件因子：业绩预告、回购增持、减持、问询、解禁、分红、停复牌
- 微观结构因子：涨停、连板、炸板、封单、竞价强度、换手强度

#### 2. 策略层

目标：

- 从“模板配置”升级为“可研究、可归因、可比较的策略对象”

建议方向：

- 模板拆分为范围层、因子层、触发层
- 模板版本化管理
- 支持因子贡献归因
- 支持不同市场环境下的不同权重与阈值
- 支持模板家族化：趋势类、事件类、修复类、轮动类等

#### 3. 板块层

目标：

- 从“先选板块”升级为“板块研究与轮动判断体系”

建议方向：

- 板块强度评分
- 板块轮动与持续性判断
- 板块内部龙头/中军/跟风识别
- 个股评分中引入板块地位与板块一致性判断

#### 4. 评估层

目标：

- 从“看短期收益”升级为“评估策略质量”

建议方向：

- 多基准评估
- 多市场环境分层评估
- 风险调整后指标
- MFE / MAE / 盈亏比 / 持有周期分布
- 按行业、市值、流动性、触发场景分层统计

#### 5. 组合层

目标：

- 从“推荐个股”升级为“形成可执行组合建议”

建议方向：

- 去相关与去重复暴露
- 单股/单板块权重约束
- 风格与行业暴露约束
- 与当前持仓组合联动
- 给出仓位、分批建仓与风险提示建议

#### 6. 执行层

目标：

- 从“研究工具”升级为“半自动执行支持平台”

建议方向：

- 模拟组合
- 纸面交易
- 券商或交易平台接入
- 交易前检查、交易后归因与执行留痕

#### 7. AI 研究副驾层

目标：

- 从“解释器”升级为“研究辅助系统”

建议方向：

- 公告/新闻/研报压缩与结构化摘要
- 候选股多空论点自动生成
- 反例挑战与风险提示
- 模板失效诊断
- 周期性复盘与风格总结

### 22.4 需要的数据源与平台能力

数据源方向：

- A 股基础行情与财务数据源
- 正式行业分类与概念分类数据源
- 公告与事件结构化数据源
- 资金流与席位数据源
- 研报、新闻、舆情与宏观事件数据源
- 指数、行业指数、风格指数与基准序列

平台能力方向：

- 特征仓库
- 模板实验平台
- 统一回测与后验评估任务
- 数据质量监控
- Prompt 版本与成本治理
- 审计与复现能力

### 22.5 最值得优先关注的专业化方向

若未来继续投入，专业价值最高的通常不是“再多几个 AI 页面功能”，而是：

- 标准化行业/概念/事件数据体系
- 多基准、多环境的评估体系
- 板块强度与轮动研究体系
- 组合风险与持仓联动体系
- AI 研究副驾与反例挑战机制

### 22.6 最终判断

- 当前版本适合作为 V1/V2/V3/V4/V5 的阶段化闭环验证路径
- V5 之后若要继续提升专业度，应优先补“数据底座、研究框架、评估体系、组合体系、平台能力”
- 未来的专业化方向不是“让 AI 做更多黑盒判断”，而是“让系统更像一套可验证、可归因、可复盘的投研平台”
